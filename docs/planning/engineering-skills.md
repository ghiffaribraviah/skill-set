# Engineering Skills Plan

This document tracks the planned engineering Daily Skills at an overview level. Individual skills will be defined in separate grilling sessions before their `SKILL.md` files are created.

## Structure

```text
skills/
  engineering/
    planning/
    workflow/
    development/
    maintenance/
```

## Planned Skills

### Planning

- `grilling`
- `prototyping`

### Workflow

- `initial-setup`
- `to-prd`
- `to-issues`
- `triage`

### Development

- `tdd`
- `gdd`

### Maintenance

- `improve-codebase-architecture`
- `documentation`
- `diagnose`

## Notes

- `.agents/skills` contains Reference Skills, not the final Daily Skills.
- `prototyping` is the Daily Skill shaped from the `prototype` Reference Skill.
- Workflow-management skills should be local-markdown-native rather than GitHub-Issues-shaped.
- `initial-setup` is planned as the repo initialization skill for configuring a target repository to use these Daily Skills.
- `gdd` and `tdd` are peer development skills with different proof models.
